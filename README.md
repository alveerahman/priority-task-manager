# Priority-Based Task Manager (DSA Focused) 📝

This is the 4th project of my **90-Day Software Engineering Sprint**. As a **Final Year CSE Student**, I developed this project to demonstrate my skills in **Data Structures, Algorithms, and Problem Solving**.

## 🚀 The Problem Solver's Approach
In real-world software, data isn't just stored—it needs to be organized. I solved the challenge of task prioritization by implementing an **Automatic Sorting Algorithm**. No matter what order tasks are entered, the system ensures High-Priority items always appear at the top.

## ✨ Key Features
- **Priority Sorting:** Uses a custom sorting key to rank tasks (High > Medium > Low).
- **Dynamic Task Handling:** Efficiently add, view, and complete (delete) tasks.
- **Data Mapping:** Uses a priority map to convert numerical levels into user-friendly status indicators.
- **Interactive CLI:** A robust command-line interface for seamless user interaction.

## 🛠️ DSA & Engineering Skills Applied
- **Sorting Algorithms:** Implemented `list.sort()` with `lambda` expressions for custom object sorting.
- **Time Complexity Awareness:** Optimized task insertion and retrieval operations.
- **Object-Oriented Design:** Encapsulated task logic within dedicated classes.
- **Error Handling:** Used `try-except` blocks to handle invalid user inputs and index errors.

## 📖 How to Run
1. Ensure you have Python installed.
2. Clone this repository: `git clone https://github.com/alveerahman/priority-task-manager.git`
3. Run: `python task_manager.py`
